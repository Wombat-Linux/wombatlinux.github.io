Comet

The package manager for Wombat Linux.

What is Comet?

Comet is the package manager for Wombat Linux. It is a simple package manager written in Rust, and is designed to be as simple as possible.

How does it work?

Comet works by using yaml files to describe packages and repositories. All packages are stored as tarballs that contain the package files and a yaml file describing the package, along with a script for installing and a script for uninstalling the package.

How do I use it?

Comet is built into Wombat Linux, so you can use it directly from the terminal. The easiest way to use it is to use the comet command, which is the main command for Comet. Here are some examples of how to use it:


  # Install a package
  comet install package1
  
  # Install multiple packages
  comet install package1 <package2> <package3>

  # Remove a package
  comet remove package1

  # Update a package
  comet update package1

  # Update all packages
  comet update-all

  # Update the package list/cache 
  comet update-cache

  # List installed packages
  comet list

  # List all packages
  comet list-available

How do I make a package?

The best way to make a package is to use the startools utility. It allows for you to create stars (packages) and build starfiles (tarballs) for them while also exporting the data to put in the repository yaml file. The startools utility is built into Wombat Linux, so you can use it directly from the terminal. The easiest way to use it is to use the startools command.
